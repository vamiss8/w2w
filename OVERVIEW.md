# Project overview

This single-page site presents two lists: titles you both still want to watch and ones you have already watched together. The UI is driven entirely by static HTML, CSS for styling/animation, and a small JavaScript file for interactions and embellishments.

## HTML structure (`index.html`)
- **`<main class="container">`** wraps all visible content. A decorative `.stars` background sits behind it.
- **Header**: `.title` shows the question (“what to watch on a date?”) and `.subtitle` lists both names.
- **Tab controls**: two hidden radio inputs (`#tab-unwatched` and `#tab-watched`) hold the state. The adjacent `<nav class="tabs">` labels toggle the radios via the `for` attributes. CSS uses the `:checked` inputs plus sibling selectors to show the matching list.
- **Lists section**: `.lists` contains two `<ul>` elements. `.unwatched` holds items without ratings; `.watched` contains items with per-person scores. Each `<li>` has:
  - a `data-status` attribute that determines the accent color in CSS.
  - a `.left` column with the title (`.filmTitle`) and a metadata line (`.meta`).
  - a `.status` badge whose color is controlled by the `s-00`/`s-01`/`s-10`/`s-11` class.
- **Footer**: a short message with crossed-heart decoration appears after the lists.
- **Script inclusion**: `main.js` is loaded at the end to animate the cards and render rating hearts.

## Styling (`style.css`)
- **Design tokens**: root custom properties define the palette for text, muted accents, and the four status colors.
- **Layout and typography**: the page centers content on a dark background, applies system fonts, and gives spacing around the container. Titles and subtitles reuse shared classes referenced in the HTML.
- **Animated starfield**: the `.stars` element and its pseudo-elements layer several `radial-gradient` backgrounds that move at different speeds via `@keyframes` animations, creating a parallax-like field of stars.
- **Tabs**: `.tabs` arranges the labels horizontally; when a radio input is checked, the matching label receives a highlighted background through sibling selectors.
- **List switching**: only the `<ul>` matching the active tab (`.unwatched` or `.watched`) is displayed (`display: grid;`).
- **Cards**: each `<li>` is a glassy card with rounded corners, gradient background, border, drop shadow, and a `fadeUp` animation on load. Hover effects lift the card slightly.
- **Ratings**: hearts generated by JavaScript inherit styles from `.rating`, `.rating-row`, `.rating-heart`, and owner-specific modifiers (`.vlad`, `.vika`, `.filled`).
- **Status badges**: `data-status` values set the border color; matching `.s-XX` classes color the badge text.
- **Footer and icons**: the footer fades in after the cards animate. Heart icons use pseudo-elements to overlap two colored hearts.

## Behavior (`main.js`)
- Selects the `<footer>` once for reuse.
- **`animateList(listSelector)`**: staggers card animations by setting incremental `animationDelay` values, hides the footer instantly, then reveals it after all cards have animated.
- Runs `animateList(".unwatched")` on first load to animate the default tab.
- **Tab persistence**: listens for changes on both radio inputs, saving the active tab id to `localStorage` and replaying the animation for the selected list. On load, the saved tab (if present) is restored and its list animated.
- **Rating rendering**: `createHearts(score, owner)` builds a set of 10 heart spans and fills those up to the provided score. `transformRatings()` scans `.watched .meta` elements, parses the "vlad: X | vika: Y" text, replaces it with two rows of colored hearts, and appends them inside `.meta`. This keeps the source HTML simple while rendering a richer rating display in the browser.

Together, these pieces create a playful, animated watchlist that remembers which list you last viewed and visualizes both partners' ratings for watched titles.
